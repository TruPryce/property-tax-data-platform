# Documentation Agent Guide

- Keep `README.md` as the landing page, `docs/README.md` as the navigation hub, and `CONTRIBUTING.md` as the developer workflow.
- Do not duplicate normative requirements from OpenSpec; link to them.
- Put stable rationale in decision records after implementation confirms it.
- End documents under `docs/` with a Related section and reciprocal links where useful.
- Use relative internal links, HTTPS external links, and language tags on fenced code blocks.
- Run `make docs` after changing maintained Markdown.
